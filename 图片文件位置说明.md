# 背景图片文件位置说明

## 📁 图片文件应该放在哪里？

### 正确的文件路径

```
项目根目录/
└── assets/
    └── images/
        └── game_background.png  ← 你的背景图片放在这里
```

### 完整路径示例

如果你的项目在 `D:\math game\`，那么图片的完整路径应该是：

```
D:\math game\assets\images\game_background.png
```

## 📝 具体步骤

### 步骤 1: 创建目录（如果不存在）

在项目根目录下创建以下目录：
- `assets/`
- `assets/images/`

### 步骤 2: 放置图片文件

1. 将你的背景图片文件复制到 `assets/images/` 目录
2. 将文件重命名为 `game_background.png`（或 `.jpg`）

### 步骤 3: 如果使用 JPG 格式

如果使用 JPG 格式，需要修改 `src/scenes/PreloadScene.js` 第 54 行：

```javascript
// 将这行：
this.load.image('game_background', 'assets/images/game_background.png');

// 改为：
this.load.image('game_background', 'assets/images/game_background.jpg');
```

## 🔍 如何验证文件位置

### 方法 1: 检查文件系统

1. 打开项目根目录（`D:\math game\`）
2. 确认 `assets/images/game_background.png` 文件存在
3. 检查文件名和扩展名是否正确

### 方法 2: 通过浏览器访问

1. 启动服务器（`npm run dev`）
2. 在浏览器中访问：`http://localhost:8788/assets/images/game_background.png`
3. 如果能看到图片，说明路径正确
4. 如果显示 404，说明文件不存在或路径错误

### 方法 3: 查看控制台

打开浏览器控制台（F12），查看：
- ✅ `✓ 资源加载成功: game_background` = 图片加载成功
- ⚠️ `⚠ 资源加载失败: game_background` = 图片未找到（但游戏仍可运行）

## 📋 文件要求

- **文件名**：`game_background.png` 或 `game_background.jpg`
- **位置**：`assets/images/` 目录下
- **格式**：PNG（推荐）或 JPG
- **大小**：建议压缩到 1MB 以下以提升加载速度
- **尺寸**：推荐 1920x1080 或更高（会自动缩放）

## ⚠️ 注意事项

1. **文件名必须完全匹配**：`game_background.png`（区分大小写）
2. **路径是相对于项目根目录**：不是相对于 HTML 文件
3. **404 错误不影响游戏**：如果图片不存在，游戏会使用渐变背景
4. **服务器需要重启**：添加新文件后，可能需要重启开发服务器

## 🛠️ 快速测试

如果你想快速测试背景功能，可以：

1. 从网上下载任意图片
2. 重命名为 `game_background.png`
3. 放在 `assets/images/` 目录
4. 刷新浏览器页面

即使图片不匹配主题，也能验证背景加载功能是否正常工作。
